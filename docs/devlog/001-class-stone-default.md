# 001 — Class Stone default highlight (open)

Lead PM is dispatching M0 (engine bones) to `coding-expert`. M2 will build
the Class Stone choice scene at Limbo Beat 8.5 (§4.8), and the UI flow
needs to know which of the three stone shapes — Sword, Staff, Shield —
appears as the default-highlighted option when the player walks up. §24.5
flags this as a marketing call: the highlight is what most first-time
players will pick, and that's the dominant default-experience the marketing
material will want to feature. The three classes do not gate ending
eligibility (§4.8), so this is purely a default-experience question, not a
balance one. Decision needed before M2 starts wiring the Class Stone scene.
M0 and M1 are not blocked. Filing this so it doesn't get lost between now
and the M2 kick-off.

**Resolved 2026-04-30:** Sword is the default-highlighted option at the
Class Stone. Rationale: aggressive duo (close-quarters with Nala flanking)
is the lowest-friction action-game default, and marketing material reads
more cleanly as "intense action game with a dog" with Sword foregrounded
than with the more cerebral Staff or the more defensive-coded Shield.
Class still does not affect ending eligibility per §4.8 — this is purely
a default-experience and marketing call. Bible §24.5 should mark this
question closed when next updated. M2 character-create can wire the Class
Stone scene with Sword as the visually-prominent center option; Staff
left and Shield right (or whichever symmetric framing the ui-ux-expert
prefers), with Sword pre-selected on scene entry.
