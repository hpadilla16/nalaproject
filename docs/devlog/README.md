# Devlog

A running record of every meaningful change to the Nala Project. One file
per entry, numbered sequentially, lowercase-hyphenated.

## Format

Filename: `NNN-short-slug.md` — `001-prologue-blockout.md`,
`002-mortality-clock-prototype.md`, etc.

Filename: `NNN-short-slug-open.md` — the `-open` suffix flags an open
question that needs a human decision. Lead PM scans for these. (`?` would
be more visually striking but isn't a legal Windows filename character;
`-open` is the portable equivalent.)

## Entry shape

One paragraph. What changed. Why. What is now possible (or blocked) as a
result. If a question is now live for the human, state it explicitly.

The tonal rules apply: no therapy-textbook language, no monologues,
brevity above all. (Bible §23.8.) Devlog entries are read by humans who
have already been doing this work for hours.
