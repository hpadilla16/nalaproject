# 008 — M1 UI/UX: Mortality HUD, Ledger HUD, and dialogue scene refactor

UI-ux-expert landed the M1 visual surfaces, replacing placeholder DebugHUD with production-ready layouts: `MortalityHUD` (`godot/scripts/ui/mortality_hud.gd` and `godot/scenes/ui/mortality_hud.tscn`) renders a vector-drawn bleeding clock with three visual states (passive bleed — calm gradient pulse; active bleed — agitated faster pulse with droplet shape when mortality is drained; critical — irregular flicker with darkened background under 10% remaining). States distinguished by gradient AND shape per §23.8 colorblind-considerateness. Listens to `MortalityClock.mortality_changed` (detects large drains as active cost) and `mortality_low`/`mortality_exhausted` signals. `LedgerHUD` (`godot/scripts/ui/ledger_hud.gd` and `godot/scenes/ui/ledger_hud.tscn`) displays two columns (Sin left, Redemption right) of identical repeated glyphs, hidden by default and toggled via Tab key (registered in `project.godot`). Small edge-flash on appropriate column when `LedgerState` emits sin/redemption events, even if panel hidden. Glyph scene at `godot/scenes/ui/ledger_glyph.tscn` (placeholder X-mark; pixel-art-sprite-expert will replace with etched rune). `DialogueRunner` scene refactored from code-built to authored tree: `dialogue_runner.tscn` now contains Panel, SpeakerLabel, and TextDisplay nodes; script wired to child nodes instead of instantiating, API unchanged (`play/advance/is_playing`, signals `line_started/line_completed/dialogue_finished`).
